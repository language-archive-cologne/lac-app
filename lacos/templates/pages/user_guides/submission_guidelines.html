The Language Archive Cologne provides technical guidance for depositors preparing data submissions.

Ethical and Legal Requirements

The archive expects researchers to respect cultural norms and reserves the right to depublish or remove data if ethical or legal concerns arise. Depositors must ensure clear, unambiguous and informed consent was collected from the speakers represented in the collection.

File Naming Recommendations

We suggest consistent naming schemes using components like:

  • Language name, glottocode, or ISO 639-3 code
  • Recording date (YYYYMMDD or YYYY-MM-DD format)
  • Running numbers with leading zeros

Example patterns: kuvi-0006.wav or samre-20090304-0023.wav. Files within a bundle should share identical names except for extensions.

Metadata Standards

Metadata should be provided in BLAM CMDI format (producible via Arbil or CMDI Maker) or as an Excel sheet template.

Format Recommendations

Audio

WAV files with LPCM encoding at 48 kHz and 16-bit depth are standard. Stereo is preferred; mono acceptable for single speakers.

Video

MOV or MP4 containers with h.264 encoding at 1080p, 30fps. MOV files support uncompressed LPCM audio; MP4 files use AAC encoding.

Annotations

ELAN format is recommended, with support for Praat TextGrid, Exmaralda, TEI, and FLeX XML.

Other Formats

PDF/A for text, TIFF/JPEG2000/PNG for images, and CSV for structured data are acceptable.
