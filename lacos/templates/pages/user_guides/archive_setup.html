Functional Architecture

The LAC employs KA3 repository software as its foundation and adheres to the OAIS reference model. The system implements open standards including Open Data Protocol (ODATA), the International Image Interoperability Framework (IIIF), OAI-PMH, and Shibboleth.

Core Components

  • Archive Information Packages (AIPs) stored in redundant file systems
  • ElasticSearch instance for metadata and descriptive information
  • Bundles as primary archival objects containing related digital files
  • Administration interface supporting bulk ingest operations

Ingest Procedure

Data submission occurs through Sciebo, SOFS (University of Cologne storage), or physical media. The process involves:

  1. Initial advisory sessions with depositors
  2. Sanity checks on files and metadata
  3. SIP conversion via Excel2BLAM or IMDI2BLAM pipelines
  4. Handle assignment and pre-OCFL structure conversion
  5. Quality verification and visual inspection
  6. Archival storage deployment

Access and Dissemination

Users access archived materials through Object and Query APIs which support a Web interface for search and presentation. The system provides downloads via web interface and metadata harvesting through OAI-PMH.

Data Management Workflow

Data moves through multiple storage locations: DCH virtual machine, LAC-dev-repository, LAC-prod instance, and ITCC Long Term Storage, ensuring preservation redundancy.

Organizational Responsibilities

  • LAC Administration - Technical coordination, strategic planning
  • LAC Management - Curation, quality assurance
  • ITCC - Infrastructure operations
  • DCH - Frontend operations
